# 打包说明

把《梦幻秒装备》打成单文件 exe，并整理出可直接发给别人的交付文件夹。

## 怎么打包
两种方式任选其一（都在本机离线进行，**不需要联网、不需要管理员**）：

- 双击本文件夹里的 **`打包.bat`**；或
- 命令行在**项目根目录**执行：`python 打包工具\build.py`

打包要一两分钟。结束后看末尾的 `[OK] 打包完成！`。

## 产出（干净平铺三件套）
交付目录名按 `mhxy/__init__.py` 里的版本号自动生成，例如版本 `2.2.0` →

```
发布\梦幻秒装备_v2.2\
    梦幻秒装备.exe      ← 单文件 exe（是文件，不是文件夹）
    config.json         ← 项目真实配置（保留全部标定，已链接 templates；dry_run 统一置演练）
    templates\          ← 随包模板图（标定 / 识别要用）
```

> ⚠ config.json 取自项目根的真实配置（保留 regions 坐标 / 各任务模板路径 / 秒装备监控清单 /
> 队长ID 历史），随包的 `templates\` 才真正链接到功能。若只发空档（旧 bug），就会出现
> 「图片还在但标定全被重置」。安全起见各任务 dry_run 统一置「演练」，用户进 GUI 一键开实战。

把整个 `梦幻秒装备_v2.2\` 文件夹发给用户即可。首次双击 exe（弹 UAC 点「是」），
同目录会自动生成 `captures\`（命中截图）。

## 本文件夹有什么
- `build.py` —— 打包脚本：调 PyInstaller + 整理交付目录（先 rmtree 推倒重建，根除同名文件夹套娃）。
- `梦幻秒装备.spec` —— PyInstaller 配置（单文件、无黑窗、manifest 请求管理员）。
  用 `SPECPATH` 锚定项目根，故脚本住在子目录也能正确找到 `start.py`。
- `打包.bat` —— 双击入口，内部调 `build.py`。
- 本说明。

## 注意
- exe 运行时才需要管理员（已写进 manifest，双击即弹 UAC），与游戏同级 SendInput 才生效；
  **打包过程本身不需要管理员**。
- `config.json` 随项目真实配置发出（含标定），故 `templates\` 是「链接好的」、开箱即用。
  注意标定坐标依赖具体屏幕/窗口尺寸（配置里记了 `targets.base_size`），换机器/换窗口大小可能要重标。
