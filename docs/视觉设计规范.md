# 视觉设计规范（UI Design System）

> 本文是《梦幻西游 · 时空》辅助助手 GUI 的**视觉基准**。后续任何界面改动都以此为准：
> 颜色、字号、间距、圆角、组件尺寸、状态色——**一律引用本文定义的令牌（token），禁止内联硬编码**。
> 本文当前为**设计基准文档**，与代码现状存在差距；差距与迁移方式见第 8、9 节。
>
> 适用范围：`mhxy/gui/` 下全部界面（主窗口 `app.py`、标定对话框 `calibrate_dialog.py`、
> 选窗对话框 `window_picker.py`、框选浮层 `roi_overlay.py`、主题 `theme.py`）。

---

## 0. 为什么要这份文档

当前界面存在明显的「不统一」与「缺能力」：

- **只有深色、不能切白天模式**：`theme.py` 是一套写死的深色十六进制色值，`app.py` 里
  `ctk.set_appearance_mode("dark")` 也写死。
- **同类元素尺寸不一**：工具按钮在「秒装备」页是 `height=34 / width=120`，在其它三个任务页是
  `height=38 / width=92`；运行按钮宽度 `180` vs `200`。
- **间距随手写**：卡片内边距 `padx=16` 与 `18` 混用，`pady` 各页不同。
- **状态色内联硬编码**：药丸（Pill）的底色 `#1d3a2b / #3a1d1d / #15301f` 直接写在 `app.py` 里，
  不在主题里，换肤换不到。
- **构建范式分叉**：任务页用 `Card()` 助手 + `grid` 布局，`GeneralPage` 自建 `_card()` + `.pack`。
- **零散硬编码**：`toast()` 里 `text_color="white"` 等。

这些会让「改一处颜色/尺寸」变成「满工程找补」。本规范用**令牌 + 组件规范 + 三段式布局**把它收敛成一套。

---

## 1. 设计原则

1. **现代、简洁**：大圆角卡片、克制的描边、充足留白、单一强调色点缀，不堆装饰。
2. **信息密度适中**：一屏能看清「在跑什么 / 状态如何 / 日志」，但不拥挤。
3. **单一事实来源**：每个语义只有一个令牌。改令牌即全局生效；组件不自带魔法数字。
4. **明暗对等**：白天/夜间是同一套语义令牌的两端取值，**不是两套独立界面**。
5. **状态用颜色说话且语义固定**：绿=安全/演练/已连接，红=危险/实战，黄=警告，蓝=主操作/强调。

## 2. 设计令牌：颜色

### 2.1 机制：令牌是 `(light, dark)` 二元组

customtkinter 的所有颜色参数（`fg_color` / `text_color` / `border_color` / `hover_color` …）
原生接受 `(浅色, 深色)` 形式的二元组，并随 `ctk.set_appearance_mode("light"|"dark")` 自动切换。

因此 **`theme.py` 的每个颜色常量都应是二元组**，例如：

```python
SURFACE = ("#ffffff", "#181c24")   # (light, dark)
TEXT    = ("#1b1f27", "#e7eaf0")
```

如此一来，**现有的绝大多数调用点无需改动**——`fg_color=T.SURFACE`、`text_color=T.TEXT`
原样就同时支持明暗。这是本规范选定的**低风险迁移路径**（详见第 9 节）。

> 约定：本文「dark 端」取值与**当前线上深色值保持一致**，保证文档落地后深色观感不变、只新增白天模式。

### 2.2 颜色令牌表

| 令牌 | light | dark | 用途 |
|---|---|---|---|
| `BG` | `#f4f5f7` | `#0e1014` | 窗口底色 |
| `SIDEBAR` | `#eceef2` | `#13161c` | 侧边栏底 |
| `SURFACE` | `#ffffff` | `#181c24` | 卡片底 |
| `SURFACE_2` | `#eef0f4` | `#20252f` | 卡片内分区 / 输入框 / 工具按钮底 |
| `BORDER` | `#dfe3e9` | `#2a313d` | 描边 / 分隔线 |
| `BTN` | `#e2e6ec` | `#2f3744` | 次级/工具按钮底（明显区别于卡片，避免糊在一起） |
| `BTN_HOVER` | `#d3d9e2` | `#3a4452` | 次级/工具按钮悬停 |
| `TEXT` | `#1b1f27` | `#e7eaf0` | 主文字 |
| `TEXT_DIM` | `#6b7280` | `#8a93a3` | 次要文字 / 说明 |
| `ACCENT` | `#2f6fed` | `#4f8cff` | 主强调（主按钮 / 选中 / 滑块 / 链接感） |
| `ACCENT_HOVER` | `#2861d8` | `#3d79ee` | 强调悬停 |
| `SUCCESS` | `#1f9d6b` | `#3ecf8e` | 成功 / 演练模式 / 已连接 / 命中(hit) |

| `ON_ACCENT` | `#ffffff` | `#ffffff` | 强调色块上的文字（主按钮 / Toast） |

> `PILL_OK_BG` / `PILL_DANGER_BG` / `ON_ACCENT` 是**新增令牌**，用来吃掉现在散落在 `app.py`
> 里的 `#1d3a2b`、`#3a1d1d`、`#15301f` 和 `"white"`。

### 2.3 日志级别色与「单值解析」

日志文本框用的是底层 tk 的 `tag_config(level, foreground=...)`，**只接受单个颜色字符串**，
不接受二元组。因此：

- `theme.py` 增加助手 `resolve(token) -> str`：按当前 `ctk.get_appearance_mode()`
  从二元组里取出对应端的单值（传入已是单值则原样返回）。
- `LEVEL_COLOR` 仍按语义定义，但配色时用 `resolve()`：

| level | 令牌 | 含义 |
|---|---|---|
| `info` | `TEXT` | 普通信息 |
| `hit` | `SUCCESS` | 命中 / 成功关键事件 |
| `warn` | `WARN` | 警告 |
| `error` | `DANGER` | 错误 / 无法启动 |

> 切换明暗后若要让已渲染的日志颜色跟随，需要重跑一遍 `tag_config`（见第 9 节迁移注意）。

## 3. 设计令牌：字体、间距、圆角

### 3.1 字体（沿用 `build_fonts()`）

字族：正文 `Microsoft YaHei UI`，等宽 `Consolas`。

| token | 字号 | 字重 | 用途 |
|---|---|---|---|
| `title` | 19 | bold | 页面 / 区块大标题、侧栏品牌名 |
| `h2` | 15 | bold | 卡片标题（监控清单 / 运行日志 / 运行参数 …） |
| `body` | 13 | regular | 正文、表单标签、按钮内文案（次级） |
| `body_b` | 13 | bold | 强调正文、列表项主名、数值 |
| `small` | 12 | regular | 说明、副标题、计数、提示 |
| `nav` | 14 | regular | 侧边导航项 |
| `btn` | 14 | bold | 主行动按钮（开始 / 停止 / 保存） |
| `mono` | 12 | regular | 日志（Consolas） |

### 3.2 间距（4 / 8 栅格）

收敛掉现在 `16` 与 `18` 混用的问题。统一令牌：

| token | px | 典型用途 |
|---|---|---|
| `SP_1` | 4 | 紧贴元素的细微间隔、行内微调 |
| `SP_2` | 8 | 控件之间的小间隔、按钮组内距 |
| `SP_3` | 12 | 卡片之间的纵向间距、列表行 pady |
| `SP_4` | 16 | **卡片统一内边距**、区块标准内距 |
| `SP_5` | 20 | 页面内容区上下外边距 |
| `SP_6` | 24 | 页面内容区左右外边距、侧栏左内距 |

固定约定：
- **卡片内边距统一 `SP_4`（16）**，废止 `18`。
- **卡片之间纵向间距 `SP_3`（12）**。
- 页面内容容器外边距 `padx=SP_6(24) / pady=SP_5(20)`（沿用 `container` 现值）。
- Body 双列之间留 `SP_3(12)`（左卡 `padx=(0,6)`、右卡 `padx=(6,0)`，二者各 6 合 12）。

### 3.3 圆角（沿用现值）

| token | px | 用途 |
|---|---|---|
| `RADIUS` | 12 | 卡片、大容器、Toast |
| `RADIUS_SM` | 8 | 按钮、输入框、下拉、列表行、小标签 |

药丸（Pill）用大圆角 `20`（胶囊形），单列为特例常量 `RADIUS_PILL = 20`。

## 4. 明暗（白天/夜间）模式

### 4.1 数据与启动

- `config.json` 顶层**新增字段 `appearance`**，取值 `"dark"` | `"light"`，默认 `"dark"`
  （`core/config.py` 的 `DEFAULT_CONFIG` 同步加默认，旧配置由 `_deep_merge` 自动补全）。
- `App.__init__` 启动时读取并应用：

```python
mode = self.cfg.get("appearance", "dark")
ctk.set_appearance_mode(mode if mode in ("dark", "light") else "dark")
```

  **替代当前写死的 `ctk.set_appearance_mode("dark")`**。

### 4.2 切换入口

- 位置：**侧边栏底部**（风险提示文案之上），一个图标按钮：夜间显示 `🌙 夜间`、白天显示 `☀ 白天`。
- 交互：点按在 `"dark"` / `"light"` 间切换 → `ctk.set_appearance_mode(新值)` →
  写回 `cfg["appearance"]` 并 `save_config` → 刷新按钮文案。CTk 控件会自动重绘成对应端颜色。
- 切换后需要手动补刷的非自动部分见 4.3。

### 4.3 必须按当前模式「解析单值」的例外清单

以下场景拿到的是底层 tk 或一次性快照，**不会随 `set_appearance_mode` 自动变**，
必须用 `resolve(token)` 取当前端单值，并在切换时重跑：

1. **日志文本框** `self.log._textbox.tag_config(level, foreground=resolve(...))`。
2. 任何**直接传给原生 tk 控件**（非 CTk）的颜色。
3. **框选浮层 `roi_overlay.py`**：纯 `tkinter.Toplevel`（为对齐 mss 截图、规避 CTk 缩放而刻意不用 CTk），
   其底色/线色需用 `resolve()` 显式取单值；它是临时浮层，通常按打开瞬间的模式渲染即可，无需热切换。

## 5. 组件规范

> 所有尺寸/颜色均引用上面的令牌。下表给的是**目标统一值**，与现状的差异见第 8 节迁移表。

### 5.1 按钮（四型）

| 型 | 底色 | 文字 | 悬停 | 尺寸 | 圆角 | 字体 | 用途 |
|---|---|---|---|---|---|---|---|
| **Primary** | `ACCENT` | `ON_ACCENT` | `ACCENT_HOVER` | 主行动 `h=46`；次级 `h=38` | `RADIUS_SM` | `btn` | 开始 / 保存 / 还原尺寸 |
| **Danger** | `DANGER` | `ON_ACCENT` | `DANGER_HOVER` | 同 Primary | `RADIUS_SM` | `btn` | 运行中「停止」 |
| **Secondary** | `BTN` + 描边 1px `BORDER` | `TEXT` | `BTN_HOVER` | **统一 `h=36 / w=104`** | `RADIUS_SM` | `body` | 选择窗口 / 标定 / 刷新配置 等工具 |
| **Ghost** | `transparent` + `border 1px BORDER` | `TEXT` | `DANGER`(删除) 或 `BORDER` | 视场景 `h≈28~30` | `RADIUS_SM` | `small` | 列表行「删除」「设为基准」「清空」 |

**关键收敛**：当前工具按钮存在 `34/120`（秒装备页）与 `38/92`（其它页）两套，**统一为 Secondary 的 `h=36 / w=104`**。

> ⚠ **必须显式设 `text_color`（白天模式踩坑）**：customtkinter 的 `CTkButton / CTkOptionMenu /
> CTkSegmentedButton` 默认文字色是**不随明暗切换的近白色**（`#DCE4EE`）——这在深色填充（Primary/Danger）上没问题，
> 但 Secondary(`BTN` 浅灰底)/Ghost(透明，浮在浅色卡片上) 在**白天模式**会变成「浅底＋近白字＝看不见」。
> 故规则：**每个按钮/下拉/分段控件都要显式给 `text_color`**——填充色块上用 `ON_ACCENT`(主/危险按钮)，
> 浅底/透明按钮用 `TEXT`。Label/Switch/CheckBox/Entry 的默认文字色本就随明暗自适应，无需显式设。

### 5.2 运行按钮状态机

| 状态 | 文案 | 底色 | 可用性 |
|---|---|---|---|
| 空闲 | `▶  开始XXX` | `ACCENT` | 正常 |
| 运行中 | `■  停止` | `DANGER` | 正常 |
| 停止中 | `停止中…` | （保持 `DANGER`） | `disabled` |

文案格式统一为 **「符号 + 两个空格 + 文案」**（`▶  ` / `■  `）。各页 `RUN_LABEL` 维持「▶  开始<任务名>」。
主行动按钮统一 `h=46`，**宽度统一**（废止 `180` vs `200`，建议 `w=200`）。

### 5.3 卡片 Card

- 统一用 `Card()` 助手：`fg_color=SURFACE`、`corner_radius=RADIUS`、`border_width=1`、`border_color=BORDER`。
- 内边距统一 `SP_4(16)`；卡内标题用 `h2` + `TEXT`，紧跟的说明用 `small` + `TEXT_DIM`。
- **废弃 `GeneralPage._card()` 自建范式**，改用 `Card()`；其布局可继续用 `.pack`，但容器与配色走统一助手。

### 5.4 药丸 Pill（胶囊标签）

圆角 `RADIUS_PILL(20)`，`padx=12 / pady=4`，字体 `small`。

| 用途 | 状态 | 底色 | 文字 | 文案样例 |
|---|---|---|---|---|
| 游戏连接 | 已连接 | `PILL_OK_BG` | `SUCCESS` | `● 号1 · 单开` / `● 3 号 · 多开` |
| 游戏连接 | 未连接 | `SURFACE_2` | `TEXT_DIM` | `○ 未检测到目标窗口` |
| 运行模式 | 演练 | `PILL_OK_BG` | `SUCCESS` | `演练` |
| 运行模式 | 实战 | `PILL_DANGER_BG` | `DANGER` | `实战` |

**所有底色走令牌**，删除 `app.py` 里的 `#1d3a2b / #3a1d1d / #15301f` 内联值。

### 5.5 其它控件

| 控件 | 规范 |
|---|---|
| **开关 Switch** | 文字 `body`；演练→实战的开关 `progress_color=DANGER`（警示）；功能性开关（如「已有宝图」）`progress_color=ACCENT`。下方配 `small/TEXT_DIM` 一行说明。 |
| **滑块 Slider** | `progress_color=ACCENT`、`button_color=ACCENT`、`button_hover_color=ACCENT_HOVER`；右侧数值用 `body_b/ACCENT`。 |
| **输入框 Entry** | `fg_color=SURFACE_2`、`border_color=BORDER`、文字 `body/TEXT`；标签 `body/TEXT` 居左。 |
| **下拉 OptionMenu** | `fg_color=SURFACE_2`、`button_color=BORDER`、`button_hover_color=ACCENT`、文字 `body`。 |
| **日志 Textbox** | `fg_color=SURFACE_2`、`text_color=TEXT`、`font=mono`、`corner_radius=RADIUS_SM`、`wrap="word"`；级别色见 §2.3。 |
| **导航项 Nav** | 字体 `nav`、`anchor="w"`、`h=42`、`corner_radius=RADIUS_SM`；未选 `fg=transparent / text=TEXT_DIM`，选中 `fg=SURFACE / text=TEXT`，悬停 `hover=SURFACE`。 |
| **列表行 Row** | `fg_color=SURFACE_2`、`corner_radius=RADIUS_SM`、`pady=SP_1(4)`；主名 `body_b/TEXT`，副信息 `small/TEXT_DIM`；右侧操作用 Ghost 按钮。 |
| **Toast** | `fg_color=ACCENT`、`text_color=ON_ACCENT`、`corner_radius=RADIUS_SM`、`padx=16/pady=8`；右下角 `relx=0.99/rely=0.97`，约 1.6s 自动消失。**`"white"` 改 `ON_ACCENT`**。 |
| **分隔线** | `fg_color=BORDER`、`height=1`，左右内边距对齐卡片内距 `SP_4`。 |
| **对话框 Dialog** | `configure(fg_color=BG)`；内部一律复用上述组件规范（标定 / 选窗对话框照此）。 |

## 6. 页面布局规范（三段式）

所有**任务页**（秒装备 / 宝图 / 运镖 / 秘境降妖）统一为三段式，从上到下：

```
┌─ Header ───────────────────────────────────────────────┐
│ 标题(title)                       [游戏药丸] [模式药丸] │
│ 副标题(small/TEXT_DIM)                                  │
├─ 控制卡片 Card ────────────────────────────────────────┤
│ [▶ 开始XXX(Primary h46)]   [选择窗口][标定][刷新配置]   │
│ ── 分隔线 ──                                            │
│ (开关 Switch + 说明)                                    │
├─ Body（grid 2:3） ─────────────────────────────────────┤
│ ┌ 左卡：运行参数/监控清单 ┐ ┌ 右卡：运行日志 ────────┐ │
│ │  ...                     │ │ [运行日志]      [清空] │ │
│ │                          │ │ ┌ Textbox ──────────┐ │ │
│ └──────────────────────────┘ └────────────────────────┘ │
└────────────────────────────────────────────────────────┘
```

布局令牌：
- 三段用 `grid`：`row=0` Header、`row=1` 控制卡片、`row=2` Body（`weight=1` 撑满）。
- Header 与控制卡片间距、控制卡片与 Body 间距均 `pady=(0, SP_3..SP_4)`。
- Body 左右列 `grid_columnconfigure(0, weight=2)`、`(1, weight=3)`，`uniform="b"`。
- 卡片标题行 `padx=SP_4`、`pady=(14, 8)`（标题）/ `(0, 12)`（内容到底边）。
- **Header 结构**：`bar` 用 grid，`col=0 weight=1`。**标题**在 `row=0, col=0`(sticky `w`)、**药丸**在 `row=0, col=1`(sticky `e`)、
  **副标题单独占 `row=1, col=0`**(sticky `ew`)。副标题不要和标题一起塞进同一个 `pack` 的左frame——那样它会被药丸挤窄、且长文本溢出。

> ⚠ **长文本必须能自动换行（踩坑：秘境降妖副标题过长被截断）**：CTkLabel **默认不换行**，单行长文本会直接被窗口右缘切掉。
> 凡是「可能超过一行」的说明性文字——**页眉副标题、卡内多行提示**——都必须：① 以 `sticky="ew"`(grid) 或 `fill="x"`(pack)
> 占满所在列宽；② 套用 `bind_wraplength(label)`（按实际宽度回流，随窗口缩放自适应）**或**显式给一个保守的 `wraplength`。
> 纯一行短词（按钮文案、药丸、表单标签）无需处理。新增页面/控件时对照本条自查。

**非任务页**：
- `GeneralPage`（通用/工具）：标题 + 可滚动卡片列，卡片走 `Card()`。
- `SettingsPage`（设置）：标题 + 可滚动卡片列（基础卡 + 速度卡），底部「保存设置」Primary。
- `AboutPage`（关于）：标题 + 单张说明卡。

## 7. 消除重复：建议抽 `BaseTaskPage`

四个任务页目前各自重复实现了几乎一致的：`_build_header`、日志卡构建、`pump`、`_log_line`、
`_clear_log`、`_render_mode_pill`、`update_game_pill`、`_open_calibrate`、运行按钮 `_toggle_run/_on_runner_finished`。

**规范建议**（后续重构方向，本次不实现）：抽一个 `BaseTaskPage(ctk.CTkFrame)` 基类，
子类只声明差异点（`TASK_NAME` / `RUN_LABEL` / 标题文案 / `need_r`、`need_t` 校验项 / 该页特有参数控件），
其余共性下沉到基类。这样新加任务页 = 填几个字段，天然符合本规范，杜绝「复制时改漏尺寸」。

> 注：这是结构性重构，单独立项；本视觉规范文档不依赖它，但它能让规范长期不被破坏。

## 9. 落地迁移指引（供后续会话执行）

按风险从低到高分步，每步可独立验证、独立回滚：

1. **令牌化 `theme.py`（地基，风险低）**
   - 颜色常量改 `(light, dark)` 二元组；dark 端 == 现有值（深色观感不变）。
   - 新增 `PILL_OK_BG / PILL_DANGER_BG / ON_ACCENT / RADIUS_PILL` 与间距常量 `SP_1..SP_6`。
   - 新增 `resolve(token)`；`LEVEL_COLOR` 配置改经 `resolve()`。
   - **回归点**：此步不改 `app.py`，启动后深色界面应与改动前**像素级一致**。

2. **配置加 `appearance`（风险低）**
   - `core/config.py` `DEFAULT_CONFIG` 加 `"appearance": "dark"`；`config.example.json` 同步。
   - 旧 `config.json` 由 `_deep_merge` 自动补全。

3. **启动应用 + 侧栏切换按钮（风险中）**
   - `App.__init__` 读 `cfg["appearance"]` 调 `set_appearance_mode`。
   - 侧栏底部加切换按钮；切换时写回配置 + 重刷日志 `tag_config`（§4.3）。
   - **回归点**：白天/夜间来回切，4 个任务页 + 通用/设置/关于 + 两个对话框都不应有「读不出的低对比」或「残留深色块」。

4. **按迁移表统一尺寸/间距/药丸/Toast（风险中，逐页）**
   - 一次改一页，改完跑一遍演练核对。重点：工具按钮 `36/104`、运行按钮 `200/46`、内边距 16、药丸令牌、Toast `ON_ACCENT`。

5. **（可选，单独立项）抽 `BaseTaskPage`（风险较高）**
   - 结构性重构，最后做；做完用「新加一个空任务页」验证复用度。

## 10. 速查（Cheat Sheet）

- 背景层级（浅→深）：`BG` → `SIDEBAR` → `SURFACE` → `SURFACE_2`。
- 文字：主 `TEXT`、次 `TEXT_DIM`、强调块上 `ON_ACCENT`。
- 语义色：蓝 `ACCENT`=主操作，绿 `SUCCESS`=安全/演练/已连接，红 `DANGER`=危险/实战，黄 `WARN`=警告。
- 圆角：卡片 `12`、按钮/输入 `8`、药丸 `20`。
- 间距：卡片内距/卡间距 `16/12`，页面外距 `24/20`，4/8 栅格。
- 按钮：主 `46`、次 `38`、工具 `36×104`、Ghost `~28`。
- 改任何颜色/尺寸：**只改 `theme.py` 令牌**，不在页面里写字面量。
